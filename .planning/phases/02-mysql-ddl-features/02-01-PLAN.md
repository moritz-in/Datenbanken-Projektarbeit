Trigger requirements (TRIG-01, TRIG-02, TRIG-03):
- Name: `trg_products_after_update`
- Event: `AFTER UPDATE ON products`
- FOR EACH ROW
- Conditional: use `IF OLD.x <> NEW.x THEN ... END IF;` for each monitored field — only log if the value actually changed
- Fields to monitor: `name`, `description`, `price`, `sku`, `load_class`, `application`, `brand_id`, `category_id`
- For each changed field: INSERT one row into `product_change_log` with:
  - `product_id = NEW.id`
  - `changed_at` (DEFAULT CURRENT_TIMESTAMP handles this — omit from INSERT)
  - `field_name = 'name'` (literal string for each field)
  - `old_value = CAST(OLD.name AS CHAR)`
  - `new_value = CAST(NEW.name AS CHAR)`
  - `changed_by = 'web_ui'`

Use `DELIMITER $$` at the top and `DELIMITER ;` at the bottom (required for multi-statement triggers in MySQL CLI init scripts). Structure:

```sql
DELIMITER $$

CREATE TRIGGER trg_products_after_update
AFTER UPDATE ON products
FOR EACH ROW
BEGIN
    IF OLD.name <> NEW.name THEN
        INSERT INTO product_change_log (product_id, field_name, old_value, new_value, changed_by)
        VALUES (NEW.id, 'name', CAST(OLD.name AS CHAR), CAST(NEW.name AS CHAR), 'web_ui');
    END IF;
    -- repeat for description, price, sku, load_class, application, brand_id, category_id
END$$

DELIMITER ;
```

Handle NULL comparison correctly: For nullable fields (description, sku, load_class, application), use:
`IF (OLD.field IS NULL AND NEW.field IS NOT NULL) OR (OLD.field IS NOT NULL AND NEW.field IS NULL) OR (OLD.field IS NOT NULL AND NEW.field IS NOT NULL AND OLD.field <> NEW.field) THEN`

For NOT NULL fields (name, price, brand_id, category_id), simple `IF OLD.x <> NEW.x THEN` is sufficient.
