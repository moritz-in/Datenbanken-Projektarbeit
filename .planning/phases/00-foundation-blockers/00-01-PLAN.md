<task type="auto">
  <name>task 2: add etl_run_log and product_change_log DDL + sku column to products</name>
  <files>schema.sql</files>
  <action>
    Insert three DDL additions into schema.sql:

    **1. etl_run_log table** — Insert BEFORE the `CREATE TABLE products` block. Use this exact DDL:

    ```sql
    -- ----------------------------------------------------------------------------
    -- Tabelle: etl_run_log
    -- Beschreibung: ETL-Lauf-Protokoll für Index-Build-Operationen
    -- ----------------------------------------------------------------------------
    CREATE TABLE etl_run_log (
        id INT NOT NULL AUTO_INCREMENT,
        strategy VARCHAR(10) NOT NULL,
        started_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
        finished_at DATETIME NULL,
        products_processed INT NOT NULL DEFAULT 0,
        products_written INT NOT NULL DEFAULT 0,
        status ENUM('running', 'success', 'error') NOT NULL DEFAULT 'running',
        error_msg VARCHAR(500) NULL,

        PRIMARY KEY (id)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
    COMMENT='ETL-Lauf-Protokoll';
    ```

    **2. product_change_log table** — Insert AFTER etl_run_log and BEFORE the trigger DDL placeholder (or before the final SELECT info block if no trigger placeholder exists). Use this exact DDL:

    ```sql
    -- ----------------------------------------------------------------------------
    -- Tabelle: product_change_log
    -- Beschreibung: Automatisches Änderungsprotokoll für Produkte (per Trigger befüllt)
    -- ----------------------------------------------------------------------------
    CREATE TABLE product_change_log (
        id INT NOT NULL AUTO_INCREMENT,
        product_id INT NOT NULL,
        changed_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
        field_name VARCHAR(100) NOT NULL,
        old_value TEXT NULL,
        new_value TEXT NULL,
        changed_by VARCHAR(100) NOT NULL DEFAULT 'system',

        PRIMARY KEY (id),

        CONSTRAINT fk_pcl_product FOREIGN KEY (product_id)
            REFERENCES products(id)
            ON DELETE CASCADE
            ON UPDATE CASCADE
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
    COMMENT='Automatisches Produktänderungsprotokoll';
    ```

    **3. sku column** — Add to the `CREATE TABLE products` definition, after the `application` column and before the closing PRIMARY KEY line:

    ```sql
    sku VARCHAR(100) NULL,
    ```

    Also add a UNIQUE constraint for sku inside the products table definition:
    ```sql
    CONSTRAINT uq_products_sku UNIQUE (sku),
    ```

    Place sku column after `application VARCHAR(50)` and before the `-- Primary Key` comment.

    **Ordering requirement:** etl_run_log must come BEFORE products (no FK dependency), product_change_log must come AFTER products (it has FK to products.id).
  </action>
