<objective>
Fix the Neo4j driver null-after-teardown bug that causes every index build to fail with
`AttributeError: 'NoneType' object has no attribute 'session'`.

Purpose: UAT Test 7 ("Index build syncs products to Neo4j — neo4j_count > 0") failed because
`@app.teardown_appcontext` fires at the end of every HTTP request (not only at process shutdown
as the original comment claimed). After any request that touches Neo4j (e.g., /rag), the hook
calls `repo.close()` → `self._driver = None`. The RepositoryFactory singleton cache keeps the
broken instance forever. The next request to /index calls `execute_cypher()` →
`self._driver.session()` → AttributeError.

Output:
- app.py: teardown_appcontext hook replaced with atexit.register (fires once at process exit)
- repositories/neo4j_repository.py: lazy-reconnect guard in execute_cypher() + close() stores
  URI/credentials for reconnect
</objective>

In `create_app()`:

1. Remove the entire `@app.teardown_appcontext` block for `_close_neo4j` (lines ~194–203).

2. Add `import atexit` to the top-level imports in app.py (alongside the existing stdlib imports).

3. Inside `create_app()`, AFTER the blueprints are registered, add an atexit-based shutdown:

```python
def _shutdown_neo4j():
    """Close Neo4j driver once at process exit (not per-request)."""
    from repositories import RepositoryFactory, Neo4jRepositoryImpl
    repo = RepositoryFactory._instances.get(Neo4jRepositoryImpl)
    if repo is not None:
        try:
            repo.close()
            log.info("Neo4j driver closed at process exit")
        except Exception:
            log.debug("Neo4j driver close skipped (already closed or never opened)")

atexit.register(_shutdown_neo4j)
```

WHY: Flask's `teardown_appcontext` fires at the end of EVERY HTTP request (the app context is
pushed per request and torn down when the request ends). `atexit` fires once when the Python
process exits — the correct semantics for a long-lived singleton driver.

Do NOT use `teardown_appcontext` for any singleton resource (DB drivers, connection pools).
`teardown_request` would be even worse (fires even earlier). `atexit` is the correct hook.

Keep all other code in `create_app()` unchanged.
